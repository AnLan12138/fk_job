### 技术选型

| 层 | 技术 | 理由 |
|---|---|---|
| 桌面壳 | Electron | 自带 Chromium，和 Playwright 共享浏览器实例 |
| 前端 | React + 简单 CSS | 状态管理直观，组件化方便 |

**关键设计：**

| 问题 | 方案 |
|---|---|
| 登录态会过期吗？ | 定期检测。投递前先访问首页检查是否已登录 |
| 被封了怎么办？ | 各平台独立 Profile，一个被封不影响其他 |
| 多账号切换？ | 暂不支持。v1 每个平台一个账号 |
| 要同时开多个浏览器吗？ | 不需要。用户授权完成后浏览器窗口关闭，投递时用 headless 模式 |

```sql
-- 简历表 (单条, 可更新)
CREATE TABLE resume (
  id INTEGER PRIMARY KEY,
  name TEXT,
  phone TEXT,
  email TEXT,
  education TEXT,     -- JSON array
  work_history TEXT,  -- JSON array
  skills TEXT,        -- JSON array
  salary_min INTEGER,
  salary_max INTEGER,
  city TEXT,
  resume_pdf_path TEXT,
  updated_at TEXT
);

-- 平台授权状态
CREATE TABLE platform_auth (
  platform TEXT PRIMARY KEY,   -- 'boss', 'liepin', 'zhilian', 'job51'
  status TEXT,                 -- 'authorized', 'expired', 'never'
  profile_path TEXT,           -- 本地 Profile 路径
  last_check_at TEXT,
  created_at TEXT
);

-- 筛选条件 (单条)
CREATE TABLE filter_config (
  id INTEGER PRIMARY KEY,
  keywords TEXT,         -- JSON: ["Java", "后端"]
  cities TEXT,           -- JSON: ["深圳", "广州"]
  salary_min INTEGER,
  experience TEXT,       -- '1-3', '3-5', '5+'
  blacklist TEXT,        -- JSON: ["外包", "996"]
  daily_limit INTEGER,   -- 每日上限
  interval_seconds INTEGER,
  updated_at TEXT
);

-- 投递记录 (核心)
CREATE TABLE delivery_log (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  platform TEXT,          -- 来源平台
  company TEXT,
  title TEXT,             -- 职位名称
  salary TEXT,            -- 薪资范围
  url TEXT,               -- 职位链接
  status TEXT,            -- 'success', 'failed', 'skipped', 'pending'
  fail_reason TEXT,       -- 失败原因
  jd_text TEXT,           -- JD 原文 (用于后续分析)
  delivered_at TEXT       -- 投递时间
);

-- 面试进度追踪
CREATE TABLE interview_progress (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  delivery_id INTEGER REFERENCES delivery_log(id),
  stage TEXT,             -- 'applied', 'hr_call', 'interview', 'offer', 'rejected'
  note TEXT,
  updated_at TEXT
);
```

## 四、投递引擎核心逻辑 (伪代码)

```javascript
// worker/engine.js

async function startDelivery(platforms, filter, resume) {
  const results = { success: 0, failed: 0, skipped: 0 };

  for (const platform of platforms) {
    const profile = await loadProfile(platform);
    if (!profile.isValid()) {
      sendToUI({ type: 'error', platform, msg: '登录过期' });
      continue;
    }

    // 搜索职位
    const jobs = await searchJobs(platform, filter);
    const deduped = deduplicate(jobs, filter.blacklist);

    // 逐个投递
    for (const job of deduped) {
      if (checkDailyLimit()) break;

      try {
        await applyOne(platform, profile, job, resume, filter);
        results.success++;
        sendToUI({ type: 'progress', platform, job, status: 'success' });
      } catch (e) {
        if (e.type === 'captcha') {
          // 验证码 → 弹窗给用户
          const solved = await promptUserCaptcha(e.image);
          if (!solved) { results.skipped++; continue; }
          await applyOne(platform, profile, job, resume, filter); // 重试
        } else {
          results.failed++;
          sendToUI({ type: 'progress', platform, job, status: 'failed', reason: e.message });
        }
      }

      // 随机延迟
      await sleep(filter.intervalSeconds + Math.random() * 5);
    }
  }

  return results;
}
```

## 五、语义匹配引擎

```
表单字段读取 → LLM 翻译 → 简历字段映射

  用户简历: 3段工作经历, 擅长Java/Spring/K8s

    "项目经历" (必填)  → 从3段经历中选最相关的:
      选"XX支付系统架构升级"那段, 因为JD提到了金融

    "技能标签" (多选)  → 自动勾选:
       勾选: Java, Spring Boot, MySQL, Redis, Kafka
       跳过: Python (JD没提)
```

## 六、安全与防封策略

| 策略 | 实现 |
|---|---|
| 真实浏览器指纹 | 使用用户自己的 Chrome 实例，不是 headless 虚拟环境 |
| 随机延迟 | 投递间隔: filter_interval + random(0, 5) 秒 |
| 每日上限 | 可配置，默认50份/天，跨平台合计 |
| 单次重试 | 失败重试3次后跳过，不无限重试 |
| 独立 Profile | 每个平台独立 Chromium userDataDir，一个被封不影响其他 |
| 无服务器 | 100% 本地运行，简历数据不上传任何服务器 |

## 十一、用户两种登录/注册场景

### 场景 A：用户已有账号

```
用户操作:
  软件里点击 [BOSS直聘] → 弹出内置浏览器
  → 页面显示 BOSS 登录页 (已有账号)
  → 用户扫码/输密码 → 登录成功
  → 浏览器关闭，登录态自动保存
  → 软件内 BOSS 状态变为 ✅ 已授权
```

### 场景 B：用户没有账号

```
用户操作:
  软件里点击 [BOSS直聘] → 弹出内置浏览器
  → 页面显示 BOSS 登录页
  → 用户点击页面上的「没有账号？去注册」
  → 在同一个内置浏览器内完成注册 (手机号+验证码)
  → 注册成功后直接进入登录态
  → 浏览器关闭，登录态自动保存
  → 软件内 BOSS 状态变为 ✅ 已授权
```

**关键点：软件不需要自己做注册功能。** 注册/登录全部在招聘网站自己的网页上完成。软件只是提供了一个内置浏览器窗口，用户在里面该怎么操作就怎么操作。完成后，软件自动把登录态存下来。

```
软件的角色:
┌─────────────────────────────────────────┐
│                                         │
│   软件 = 一个「浏览器壳子」+「储柜」     │
│                                         │
│   用户打开内置浏览器 → 去招聘网站        │
│   用户自己登录/注册 (跟用Chrome一样)     │
│   登录完成后 → 软件偷偷保存 cookie/session│
│   下次投递 → 直接从储柜取出来用          │
│                                         │
│   用户不需要知道 cookie 是什么           │
│   用户只看到: 登录了一次，以后都能用      │
│                                         │
└─────────────────────────────────────────┘
```

## 十二、关于「拟人投递」的精准回答

你问「后端有没有办法通过拟人的方式去投递，现在的方式是什么」。

### 现在的方式 = 就是拟人投递

```
不是:
  ❌ 调 API 接口
  ❌ 伪造 HTTP 请求
  ❌ 爬虫抓包

而是:
  ✅ Playwright 操控真实 Chrome 浏览器
  ✅ 跟真人在浏览器里操作一模一样
  ✅ 鼠标点击 → 键盘输入 → 翻页 → 提交
  ✅ 使用用户自己的 Chrome 指纹 (userDataDir)
  ✅ 随机延迟 3-8 秒模拟人类操作节奏
```

### 具体怎么「拟人」

| 真人操作 | Playwright 模拟 |
|---|---|
| 打开浏览器，输入网址 | `page.goto('https://www.zhipin.com')` |
| 在搜索框打字「Java」 | `page.fill('input.search', 'Java', { delay: 100 })` ← 逐字打，有间隔 |
| 鼠标移到「城市」下拉框 | `page.hover('.city-selector')` |
| 点选「深圳」 | `page.click('text=深圳')` |
| 选「3-5年经验」 | `page.click('[data-value="3-5"]')` |
| 滚动页面看列表 | `page.evaluate(() => window.scrollBy(0, 500))` |
| 点进职位详情 | `page.click('.job-card:first-child')` |
| 读 JD 内容 | `page.textContent('.job-detail')` |
| 填「自我介绍」 | 从简历取 + LLM 生成 → 逐字输入 |
| 上传简历文件 | `page.setInputFiles('input[type=file]', 'resume.pdf')` |
| 点「投递」按钮 | `page.click('button:has-text("立即沟通")')` |
| 等一会儿 | `sleep(random(3000, 8000))` |
| 点下一个 | 循环 |

### 和真人的唯一区别

| | 真人 | 这个软件 |
|---|---|---|
| 操作者 | 你的手 + 眼睛 | Playwright 脚本 |
| 速度 | 时快时慢 | 编程控制，可快可慢 |
| 疲劳 | 投 10 个累死 | 投 100 个无感 |
| 判断力 | 人眼看 JD 判断 | LLM 分析 JD 匹配度 |
| 登录 | 你手动登 | 你登一次，后续自动 |
| 验证码 | 你自己滑 | 弹窗给你滑 |

**本质上 = 一个不累的你，24 小时帮你投。**
